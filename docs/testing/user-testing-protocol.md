# Protocolo de Pruebas con Usuarios

## Objetivo

Evaluar usabilidad y comprension de flujos clave del sistema en estudiantes, tutores y coordinadores.

## Participantes

- Perfil A: Estudiantes en internado
- Perfil B: Tutores clinicos
- Perfil C: Coordinadores

Tamano recomendado:

- Minimo 8 participantes (ideal: 12)

## Escenario controlado

- Navegador actualizado
- Ambiente de pruebas con datos semilla
- Sesion de 20-30 minutos por participante

## Guion de tareas

1. Iniciar sesion y validar acceso al panel correspondiente al rol.
2. Registrar una asistencia diaria.
3. Registrar una entrada de bitacora semanal.
4. Reportar un incidente.
5. Consultar estado o resultados del flujo segun rol.

## Instrumentos de captura

- Exito por tarea (0/1)
- Tiempo por tarea (opcional)
- Escala de satisfaccion 1-5
- Comentarios cualitativos

## Criterios de exito academico

- Tasa de completitud por tarea >= 80%
- Satisfaccion promedio >= 4.0
- Comentarios criticos documentados con accion de mejora

## Archivo de registro recomendado

Usar el formato CSV de `docs/testing/data/user-tests-simulated.csv` como plantilla base para ejecuciones reales.
